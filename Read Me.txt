El formulario es un container que tiene un input por campo(nombre,email,contraseñ y confirmar contraseña). Abajo esta el botón de tipo sumbit "enviar respuesta" y si lo clickeas no te deja ir a la siguiente página sin que los datos estén correctos. Arriba de eso esta el botón mostrar contraseña que justamente muestra los dos campos de la contraseña si clickeas el botón y luego los oculta si clickeas de nuevo.

Para validar los datos, creamos contenedores tanto si la información puesta está bien como si está puesta mal. Estos contenedores los inicializamos sin nada y si no se cumplen las condiciones( como que el input de usuario tenga 3 caracteres). Va a aparecer un texto rojo correspondiente. Encambio si está bien, va a aparecer un texto en verde. Luego para mostrar el mensaje de validación, este ya está escrito pero está invisible(gracias a display:none) hasta que se completen correctamente todos los campos. Luego de que aparezca el mensaje de validación se tarda 2000 milisegundos en redirigirse a otra página donde se agradece el envío del formulario. También validamos que las dos contraseñas coincidan luego de chequear que la primera cumpla con las condiciones requeridas. 

Como gran desafio, tuvimos el problema de que se muestren y oculten las contraseñas, para eso, utilizamos query selector para elegir los contenedores de ambos inputs y luego dependiendo si se queria mostrar o esconder, se cambia el tipo de input del contenedor(de text a password o al revés).
